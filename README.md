# 🗺️ WayGo Map System

## 📌 Опис

Цей модуль відповідає за роботу з картою у проєкті **WayGo** (таксі та доставка).

Він дозволяє:

* відображати карту
* знаходити адреси
* будувати маршрут між точками
* працювати з координатами (lat/lon)

---

## ⚙️ Технології

### 1. Leaflet

**Роль:** логіка карти

Використовується для:

* створення карти (`L.map`)
* роботи з координатами (`L.latLng`)
* додавання маркерів (`L.marker`)
* керування масштабом та позицією (`setView`, `fitBounds`)

👉 Простими словами:
**Leaflet — це “движок” карти**

---

### 2. OpenStreetMap

**Роль:** відображення карти

Використовується як джерело:

* доріг
* будинків
* міст

Підключається через:

```javascript
L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png')
```

👉 Простими словами:
**OpenStreetMap — це “сама карта (зображення)”**

---

### 3. Leaflet Routing Machine

**Роль:** побудова маршруту

Використовується для:

* прокладання маршруту між точками
* відображення дороги на карті

Приклад:

```javascript
L.Routing.control({
    waypoints: [start, end]
})
```

👉 Простими словами:
**Це “навігатор” як у Google Maps**

---

### 4. Nominatim

**Роль:** пошук адрес

Використовується для:

* перетворення тексту в координати
* автодоповнення адрес

Приклад запиту:

```javascript
https://nominatim.openstreetmap.org/search
```

👉 Простими словами:
**Це “пошук адрес”**

---

## 🔄 Як це працює разом

1. Користувач вводить адресу
2. Nominatim знаходить координати
3. Дані передаються в Leaflet
4. Leaflet відображає карту
5. OpenStreetMap підтягує зображення карти
6. Leaflet Routing будує маршрут

---

## 📊 Схема

```
Користувач → Вводить адресу
        ↓
Nominatim → координати (lat/lon)
        ↓
Leaflet → створює карту
        ↓
OpenStreetMap → підвантажує карту
        ↓
Routing → будує маршрут
```
